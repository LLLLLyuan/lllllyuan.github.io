AcWing790题——数的三次方根(二分查找)
  • Oct 27, 2019
  • 2 minutes read
  • This content have been read times.

题目

给定一个浮点数n，求它的三次方根。

输入格式

共一行，包含一个浮点数n。

输出格式

共一行，包含一个浮点数，表示问题的解。

注意，结果保留6位小数。

数据范围

−10000 ≤ n ≤ 10000

输入样例：

1000.00

输出样例：

10.000000

题目解读

像这种求平方根、立方根等等，是典型的二分查找，需要注意的是题目的精度问题。

解题思路

1、确定开始的分界点，分为左右两个区域。

2、判断分界点是否符合条件，若符合递归，若不符合换个区间递归

解题步骤

确定分界点

假设有一段左右两端为 l 、 r，的区间，则mid = (l+r)/2

l          mid           r
├——————————-┤————————————┤

因为是浮点数二分，不用管边界问题，每次 mid 都取（l+r）/ 2。

判断并更新区间

我们已知，初始时l = 0，r = 10 ，我们要求27的立方根，我们知道27的立方根是3，如下图

l   3    mid=5          r
├———┤—————-┤————————————┤

此时我们可以判断\(mid^3\) = 75 ，大于 27，那么下一次递归区间应该在[l,mid]之间。

同理，若 \(mid^3\) < 27 ，那么下次递归区间就应该是在[mid,r]之间。

一直递归，直到递归的精度达到要求，停止递归。

代码实现

#include <iostream>

using namespace std;

int main()
{
    double x;
    cin >> x;

    double l = -100, r = 100;   //题目输入的数字范围为−10000 ≤ n ≤ 10000 ,±100足够了，但是不能太小
    while (r - l > 1e-8)   //若左右两边区间的精度范围达到1e-8，此时精度完全足够，一般要比题目多2位
    {
        double mid = (l + r) / 2;   //递归取中点
        if (mid * mid * mid >= x) r = mid;  // 判断 x应在mid的哪一边
        else l = mid;
    }

    printf("%.6lf\n", l); 
    return 0;
}

这个相较于整数二分而言非常简单，主要是不需要考虑边界问题，浮点数可以直接除，没有向下取整这个问题。

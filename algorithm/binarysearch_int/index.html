AcWing789题 ——数的范围(二分查找)
  • Oct 27, 2019
  • 4 minutes read
  • This content have been read times.

先看题

题目

给定一个按照升序排列的长度为n的整数数组，以及 q 个查询。

对于每个查询，返回一个元素k的起始位置和终止位置（位置从0开始计数）。

如果数组中不存在该元素，则返回“-1 -1”。

输入格式

第一行包含整数n和q，表示数组长度和询问个数。

第二行包含n个整数（均在1~10000范围内），表示完整数组。

接下来q行，每行包含一个整数k，表示一个询问元素。

输出格式

共q行，每行包含两个整数，表示所求元素的起始位置和终止位置。

如果数组中不存在该元素，则返回“-1 -1”。

数据范围

1 ≤ n ≤ 100000 1 ≤ q ≤ 10000 1 ≤ k ≤ 10000

输入样例：

6 3
1 2 2 3 3 4
3
4
5

输出样例：

3 4
5 5
-1 -1

题目解读

这道题的意思就是，我输入一个有序数组n，然后要查询q个数，你给我返回每个要查询的数的起始位置和结束位置(相当于第一次出现和最后一次出现的位置)，没有的话就返回 -1 -1。

比如举例的[1, 2, 2, 3, 3, 4]，我要查2，那么结果应该返回 1 2（返回数组中的index），我要查5，数组里没有，那就返回-1 -1。

当然，如果不考虑时间的话，从数组第一个一直找到最后一个，这样是可以的，但是当数组非常大的时候，且查找的数非常靠后的话，耗费的时间就会非常久了，顺序查找平均时间复杂度为O(n)。

因为这道题是顺序数组，具有单调性，所以这里可以使用二分查找的方法，二分查找效率较顺序查找高，时间复杂度为O(\(log_2{n}\)）。

首先要补充一点的是，有单调性的题目可以进行二分，但是可以二分的题目不一定有单调性。

解题思路

1、确定开始的分界点，分为左右两个区域。

2、判断要查找的数是否在左边，是的话递归循环找下去，找到最左边的那个要找的数。

3、判断要找的数是否在右边，是的话也递归循环找到最右边的那个要找的数。

解题步骤

确定分界点

假设有一段长为 n，左右两端为 l 、 r，的数组

l          mid           r
├——————————-┤————————————┤

因为是二分查找，又是整数二分所以刚开始的时候，所以中间值每次为 mid = (l + r) / 2 。

判断并更新区间

我们已知，初始时l = 0，r = n-1

l   x     mid           r
├———┤—————-┤————————————┤

①假设 x 在左边的区间，即 mid >= x 时，第一次更新区间时规则如下：

mid = (l + r)/2;
//若 mid >= x ，则 r = mid，l = 0 不变，新的区间为[l , mid]
if mid >= x , r = mid;
// 若不满足，则 l = mid + 1 , r = n-1 不变，新的区间为[mid + 1 , r]
else l = mid + 1;

之后只用判断 l＜r 是否成立，如果成立则继续循环，直到找到最左边的x的位置

l          mid         x   r
├——————————-┤——————————┤——┤

②假设 x 在右边的区间，即 mid <= x 时，第一次更新区间时规则如下：

mid =（l + r + 1）/ 2;
// 若 mid <= x ，则 l = mid，r = n-1 不变，新的区间为[mid , r]
if mid <= x , l = mid;
// 若不满足，则 r = mid - 1 , l = 0 不变，新的区间为[l , mid - 1]
else r = mid - 1;

这里有个边界问题，mid =（l+r+1）/ 2 是因为这是整数二分，会向下取证，mid若取 (l+r)/2 会发生边界问题，一直递归。

比如l=0,r=1，mid=(0+1)/2向下取整为0，还是[0,1]，会无限递归下去

之后也是判断 l<r 是否成立，然后递归找到最右边的x的位置

代码实现

#include <iostream>

using namespace std;

const int N = 100010;

int n, m;
int q[N];

int main()
{
    scanf("%d%d", &n, &m); //输入数组长度与要查询的次数
    for (int i = 0; i < n; i ++ ) scanf("%d", &q[i]);  //输入数组

    while (m -- )  //  循环查询，也可以用for循环  for ( ; m>0; m--) 
    {
        int x;
        scanf("%d", &x);       //输入查询的数

        int l = 0, r = n - 1;       
        while (l < r)         //此处是第一个二分，也就是查找最左边的x，结束标志位l =r 
        { 
            int mid = l + r >> 1;
            if (q[mid] >= x) r = mid;   //判断是否符合x在mid的左边
            else l = mid + 1;          //若不是则取右边，l = mid +1 
        }
        
        if (q[l] != x) cout << "-1 -1" << endl;   //二分的结果应该是q[l]==x 若不是返回-1 -1
        else   //if判断的q[l]也可以换成q[r]，因为二分的结束标志就是l=r，即区间不能再分
        {
            cout << l << ' ';   //输出最左边的x的位置

            int l = 0, r = n - 1;
            while (l < r)    // 二分查找最右边的x的位置,也可以for循环 for (;l<r;)
            {
                int mid = l + r + 1 >> 1;
                if (q[mid] <= x) l = mid;
                else r = mid - 1;
            }

            cout << r << endl;  //输出最右边x的位置
        }
    }

    return 0;
}

写的解题步骤可能不是很清晰，看起来有点晕，其实画个图就清晰多了，这个题要用两次二分找到x的起始位置和结束位置，只要理解了当 mid <= x 时，mid为什么要是（l+r+1）/2 而不能是 （l+r）/2，这样其他的就会都明白了。

如果还是不清楚的话可以先看另一篇浮点数二分，然后再拐回来看整数二分。
